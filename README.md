# ZooWorld

## Краткое описание
- Спавн новых животных каждые 1–3 секунды.
- Правила пищевой цепи: хищник ест жертву; хищники сражаются с вероятностью победы из конфига.
- Физические столкновения и удержание в пределах видимой области.
- Всплывающий текст «Tasty!» при поедании.
- HUD со счётчиком погибших жертв и хищников.

## Основные сущности
- **Животные**: имеют роль в пищевой цепи (`Prey`/`Predator`), движение (прыжок или равномерное перемещение), поведение при столкновении.
- **Движение**: стратегии `JumpMovementBehavior` и `WanderMovementBehavior`, создаются фабрикой.
- **Столкновения**: `AnimalCollisionResolver` применяет правила пищевой цепочки и отправляет сигналы.
- **Спавн**: `AnimalSpawner` выбирает животных по весам из `AnimalCatalog` и размещает в границах `WorldBoundsService`.
- **UI**: `GameStatsModel` + `GameHudViewModel` + `GameHudView` показывают статистику.
- **Эффекты**: `TastyTextPool` создаёт/переиспользует всплывающий текст.

## Архитектура и технологии
- Zenject (SignalBus, фабрики, инсталлеры).
- MVVM для UI слоя.
- Паттерн стратегия для движения, централизованный обработчик столкновений.

## Настройка
- `GameConfig`: интервалы спавна, дистанция камеры, паддинг границ, вероятность победы хищника, высота текста «Tasty!».
- `AnimalCatalog`: список животных с их ID, диетой, префабом, весом спавна и типом движения.


## Расширение
1) Добавьте новый `AnimalDefinition` в `AnimalCatalog` (ID, диета, префаб, движение).  
2) При необходимости создайте новую стратегию движения и зарегистрируйте её в `MovementBehaviorFactory`.  
3) Настройте веса спавна и параметры движения в инспекторе.
